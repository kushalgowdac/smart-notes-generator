## Understanding `this` pointer

*   The **`this` pointer** is a fundamental concept in object-oriented programming.
*   **Problem Introduction:**
    *   When an **object** of a **class** is created, each object has its own copy of **data members**.
    *   However, **member functions** are common to all objects of that class.
    *   If multiple objects call the same member function, a key question arises: How does that function distinguish which specific object's data it should operate on?
*   **Solution:** The `this` pointer is utilized to address this problem.

> **Definition:** The `this` pointer is an **inbuilt** (or **implicit**) pointer that consistently holds the memory address of the **current object** through which a member function has been invoked.

*   **Characteristics of `this` pointer:**
    *   It is automatically available within every **non-static member function**.
    *   It directly indicates the object that initiated the member function call.
    *   It is a **constant pointer**, meaning its value (the address it stores) cannot be re-assigned after its initial setup.
